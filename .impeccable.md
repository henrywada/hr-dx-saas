# Impeccable Design Context — HR-DX SaaS

## Design Context

### Users

三つのロール：

1. **一般従業員**（最多ユーザー） — ストレスチェック回答、QR出退勤、サーベイ参加。スマホ・PC混在。業務の合間に素早く完了したい。不安や抵抗感なく使えることが最重要。
2. **HR管理者（テナント管理者）** — 人事担当者。PCメイン。データを見て判断し、設定を変更する。情報の密度と視認性が重要。
3. **SaaS運営者** — テナント管理・サービス設定・統計把握。PCオンリー。プラットフォーム全体を俯瞰するオペレーター視点。

**コンテキスト**: 日本のオフィス環境、業務時間中の使用。明るい環境での操作がメイン。

### Brand Personality

**信頼 × 効率 × 明快**

- 信頼：揺るがない安定感、誠実さ、データを正確に扱う
- 効率：無駄を削ぎ落とした操作性、情報への最短経路
- 明快：迷わない構造、はっきりした階層、説明不要のUI

感情的ゴール：「このシステムなら安心して使える」「情報が一目でわかる」「操作に迷わない」

### Aesthetic Direction

**洗練されたミニマル** — Linear / Vercel / Apple に近い精度感。ただし日本語コンテンツを主体とした温かみのある実用性。

- **テーマ**: ライトモード（オフィスの明環境、業務ツールとしての信頼感）
- **参照**: Linear（精度感）、Notion（親しみやすさ）、SmartHR（日本HRの先行事例）
- **アンチリファレンス**: 量産型SaaSダッシュボード（同じ大きさのカードが並ぶやつ）、Salesforce的な重厚さ、generic AI UI

**カラー戦略**: プライマリブルー #0055ff を軸に OKLCH で再構築。ニュートラルはブルーに向けてわずかにティント。Teal/Orangeはアクセントとして稀に使う（60-30-10原則）。

**タイポグラフィ**: Noto Sans JP の可読性を活かしつつ、英数字・見出しには個性のある欧文フォントを組み合わせる。日本語本文は固定スケール（アプリUIのため）、英数字の数値表示には tabular figures で整列。

### Design Principles

1. **情報の重力** — 重要な数値・状態は大きく、補足情報は小さく。均一な強調は強調ではない。
2. **ホワイトスペースは機能する** — 余白を詰め込みで埋めない。呼吸できる空間が信頼感を生む。
3. **カードは最終手段** — すべてをカードで囲まない。必要なときだけ境界を引く。テーブル、インライン表示、セクション区切りを先に検討する。
4. **日本語ファースト** — UI文言は日本語を基準に設計する。英字は補足・ラベル・数値に限定。
5. **状態を明示する** — ローディング、エラー、空状態、成功、注意 — それぞれ視覚的に区別できるデザイン。
6. **統一感は変数から生まれる** — spacing, color, radius, shadow をトークンで統一。コンポーネントの見た目の統一より、システムの一貫性を優先する。

### What to Avoid

- `border-left: Xpx solid [color]` のサイドストライプカード（量産型の象徴）
- グラデーションテキスト
- アイコン+大きい数値+小ラベルの「ヒーローメトリック」テンプレートをそのまま使う
- 全要素に同じ `rounded-xl shadow-sm` のカードを適用
- 「とりあえずダークモード」
