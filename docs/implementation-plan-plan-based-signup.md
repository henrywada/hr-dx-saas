# 実装計画: LP料金プラン連動セルフサインアップ（プラン別基本設定）

作成日: 2026-07-16

## 1. 問題定義

LP（https://hr-dx.jp/pricing/）の料金プランページは現在「Coming Soon」のままで、
見込み顧客がプランを選択して SaaS（https://app.hr-dx.jp）の利用を開始する導線が存在しない。
また SaaS 側の既存セルフサインアップ（`src/features/signup/`）はテナント作成後に
利用可能サービス（`tenant_service`）を一切付与しないため、登録直後のテナントは
メニューに機能が表示されない状態になる。

## 2. ユーザーストーリー

- 見込み顧客（中小企業の経営者・人事責任者）として、LP の料金ページでプランを比較し、
  無料プランをその場で申し込んで即日利用を開始したい。
- 申込者として、会社名・業種・氏名・会社メールアドレスを入力するだけで、
  自社専用テナントと管理者アカウントが自動作成され、招待メールを受け取りたい。
- SaaS 運営者として、プランごとに有効なサービスセットをテンプレートとして管理し、
  新規テナントへ自動適用したい。

## 3. 要求と優先度

| #   | 要求                                                                                                                      | 優先度                     |
| --- | ------------------------------------------------------------------------------------------------------------------------- | -------------------------- |
| R1  | 料金プランを5種に刷新: 無料(〜30名) / プラン100(〜100名) / プラン300(〜300名) / プラン500(〜500名) / プラン1000(〜1000名) | MUST                       |
| R2  | 今回は無料プランのみ有効。他4プランは LP・SaaS 双方で「準備中」表示                                                       | MUST                       |
| R3  | 入力項目: 会社名(必須)・業種(任意)・氏名(必須)・メールアドレス(必須)                                                      | MUST                       |
| R4  | メールアドレスは会社メールのみ許可（フリーメールドメイン拒否リスト方式、クライアント＋サーバー両検証）                    | MUST                       |
| R5  | テナント作成時、プラン名と同名のテンプレートテナント（PlanFree 等）の tenant_service を新テナントへコピー                 | MUST                       |
| R6  | 管理者ユーザ作成（auth.users + employees, app_role='hr'）と招待メール送信                                                 | MUST（既存流用）           |
| R7  | 有料プランの Stripe 決済                                                                                                  | 将来（コード構造のみ準備） |

## 4. データモデル

### 変更（マイグレーション `20260716*_plan_rename_and_templates.sql`）

- `tenants.is_template BOOLEAN NOT NULL DEFAULT false` — テンプレートテナント識別フラグ
- `tenants.industry TEXT` / `tenant_contracts.industry TEXT` — 業種（任意入力）
- `plan_type` の値刷新: `'free'|'plan100'|'plan300'|'plan500'|'plan1000'`
  （既存データ移行: `pro`→`plan100`、`enterprise`→`plan1000`）
- テンプレートテナント5件を固定 UUID で INSERT（name = `PlanFree` / `Plan100` / `Plan300` / `Plan500` / `Plan1000`、`is_template=true`）

### テンプレートテナントの運用

- tenant_service の中身（プランごとの有効サービスセット）は SaaS 管理画面
  （システムマスタ > テナント×サービス）から手動設定する。
- テンプレートは契約テナント一覧（/saas_adm/tenants）には表示しない。削除も禁止。

## 5. 配置ルール（LP / SaaS の責務分担）

| 責務                                           | 配置                      | 理由                                                                               |
| ---------------------------------------------- | ------------------------- | ---------------------------------------------------------------------------------- |
| プランカード表示・CTA リンク                   | LP（hr-dx-lp, Astro SSG） | 静的サイトのため。秘密情報・フォーム処理を持たない                                 |
| 入力フォーム・検証・決済・テナント作成・メール | SaaS（hr-dx-saas）        | Server Actions + service_role による安全なブートストラップ。既存 signup 基盤を流用 |

LP → SaaS の連携は `{PUBLIC_APP_URL}/signup?plan=free` へのリンクのみ（API 連携なし）。

## 6. マスタ登録（リリース前の手動作業）

1. 本番 Supabase へマイグレーション適用（コードデプロイと同時。事前確認必須）
2. システムマスタ画面で各テンプレートテナント（まず PlanFree）に有効サービスを割当
3. LP 本番ビルド環境に `PUBLIC_APP_URL=https://app.hr-dx.jp` を設定

## 7. 成功指標

- LP /pricing から `/signup?plan=free` 経由でテナント登録が完了し、
  初回ログイン時に PlanFree 相当のサービスメニューが表示される
- フリーメール（gmail.com 等）での登録がクライアント・サーバー両方で拒否される
- `/signup?plan=plan100` 等の直接アクセスで「準備中」画面が表示され、登録 API も拒否する

## 8. オープンクエスチョン

- 有料プランの価格・Stripe Price ID（準備中のため未定。`PLAN_CONFIG` の
  `available` / `stripePriceIdEnv` 切替で開通する構造とする）
- テンプレートテナントのサービスセット初期値（運用で決定）
- 管理画面の手動テナント作成（tenant-management/createTenant）への
  tenant_service コピー適用は今回スコープ外（将来対応）
