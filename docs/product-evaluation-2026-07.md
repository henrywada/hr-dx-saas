# hr-dx-saas プロダクト総合評価（2026年7月）

- 作成日：2026-07-08
- 作成方法：コードベース全機能の棚卸し（src/features 52ドメイン・app ルート群）＋ Web 調査（競合製品・法改正・HR テックトレンド）
- 評価観点：(1) 日本の中小企業（従業員100〜1000名）の経営者・人事責任者ニーズへの適合度、(2) 競合比較、(3) 社会動向を踏まえた追加機能

---

## 評価サマリー

- **プロダクトの2大ゴール（コミュニケーション促進・組織健康度の可視化）への適合度は高い。** ストレスチェックは厚労省57項目準拠（男女別の素点→評価点換算まで実装）で、高ストレス者判定→産業医面談フォロー→労基署報告書まで一気通貫。タレントマネジメント（評価/360度/OKR/1on1/スキルマップ/サクセッションプラン）も揃っている。
- **構造的な弱点は「労務実務ゼロ」「外部連携ゼロ」「実績・信頼性資産の不足」の3点。** 給与計算・年末調整・社会保険手続きがないため SmartHR 等との併用が前提になるが、freee/SmartHR/Slack 連携も公開 API もないため従業員マスタの二重入力が発生する。
- **追い風の法改正が2つ控えている。** カスハラ対策義務化（2026年10月施行）と50人未満事業場のストレスチェック義務化（2028年4月施行）。いずれも既存資産（consultation 基盤・ストレスチェック中核）をそのまま活かせるため、最優先の投資先。

## 1. 中小企業（100〜1000名）ニーズへの適合度

中小企業白書・帝国データバンク調査によると、中小企業の最大の経営課題は「人材確保」であり、採用難（「募集しても応募がない」30.6%）、定着率の低さ、賃金水準のミスマッチが上位を占める。

### よく応えられている領域

| ニーズ                                  | 対応する実装                                                                                                                                         |
| --------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------- |
| メンタルヘルス法令対応                  | 57項目準拠ストレスチェック、高ストレス者判定、産業医面談フォロー（company_doctor）、労基署報告書（gov-report）、集団分析、同意管理                   |
| 定着・離職防止                          | 離職リスク分析（turnover-risk）、パルスサーベイ、1on1（AI要約付き）、kudos、エンゲージメント、退職面談（exit-interview）、コンディションチェックイン |
| 採用難                                  | AI求人票生成（recruitment-ai / job-postings）、ハローワーク連携、リファラル、採用ファネル、内定検証                                                  |
| 100名超で必要になるタレントマネジメント | 評価（通常/360度/期間/テンプレ/ワークフロー）、OKR、スキルマップ、サクセッションプラン、キャリア面談                                                 |
| 低予算・IT部門なし                      | AI機能フリーミアム（無料プラン月10回、`try_consume_ai_usage` RPC で原子的消費）+ Stripe 決済、Server Actions 中心のシンプル構成                      |
| 勤怠の多様性                            | QR / PC / CSV の3打刻方式、36協定分析、締め処理                                                                                                      |

### ギャップ（応えられていない点）

1. **労務実務が皆無** — 給与計算・年末調整・社会保険手続き・電子申請（e-Gov）・雇用契約・マイナンバー管理はゼロ（hr-assistant で「相談できる」、lifecycle で「タスク名として管理できる」のみ）。中小人事の最大工数領域であり、SmartHR / freee との併用が前提になる。
2. **外部連携・公開 API がない** — 連携は Stripe（決済）とメール（SMTP）のみ。併用前提なのに従業員データの二重入力が発生する。
3. **多言語なし・PWA/ネイティブアプリなし** — 外国人雇用の増加、現場系（非デスクワーク）従業員のサーベイ回答率に不利。
4. **スタブ状態のドメインが残存** — telework / labor-compliance / survey / market-analysis / offer-validation 等は片側実装。機能カタログと実態の乖離は商談で信頼を損なうリスク。
5. **myou（在庫トレーサビリティ）の混在** — HR 製品としてプロダクトフォーカスがぼける。分離またはポジショニングの整理が必要。

### 結論

ニーズの「攻め」側（採用・定着・組織健康）には強く応えているが、「守り」側（労務効率化）は意図的にカバー外。**「守り（労務）は他社、攻め（組織健康・タレマネ・採用）＋法定メンタルヘルスは hr-dx」というポジショニングを明確化し、外部連携でつなぐ**のが現実的な戦い方。

## 2. 競合比較

| 競合                                  | 相手の強み                                     | hr-dx との関係                                               |
| ------------------------------------- | ---------------------------------------------- | ------------------------------------------------------------ |
| SmartHR                               | 労務手続きシェア No.1、60,000社以上導入        | 労務では戦えない。タレマネ・サーベイへ拡張中で将来的な脅威   |
| カオナビ / HRBrain / タレントパレット | タレントマネジメント専業、評価・配置分析の深さ | 評価/OKR/スキルマップは機能重複大。分析深度・導入実績で劣後  |
| Wevox / ラフールサーベイ / ミキワメ   | 組織サーベイ専業、Wevox は1人月600円と安価     | パルス＋法定ストレスチェックの統合が差別化点                 |
| Carely / WELSA 等の健康管理システム   | 産業保健の一気通貫、健康診断管理               | **実は最も近い競合**。健診管理・休職復職管理で先行されている |

### hr-dx の強み

1. **オールインワン** — 法定ストレスチェック＋タレマネ＋採用＋勤怠＋eラーニングを1製品で提供。中小企業はツール乱立と複数契約を嫌う。
2. **AI の横断活用（Gemini）** — 求人生成、職場改善提案（ストレスチェック実施マニュアル第8章準拠のプロンプト設計）、1on1要約、eラーニング自動生成、RAG 人事相談チャット。専業競合はここまで横断的でない。
3. **価格の柔軟性** — フリーミアム＋プラン別制限（free/pro/enterprise）を実装済み。
4. **マルチテナント RLS による厳格なテナント分離**（アーキテクチャ上の信頼性）。

### hr-dx の弱み

1. 導入実績・事例・ブランドの不在（比較サイトに載らない製品は中小の検討土俵に乗らない）。
2. 外部連携ゼロ（競合はすべて主要労務・給与ソフトと連携済み）。
3. セキュリティ認証（ISMS / プライバシーマーク等）— 人事データを預かる SaaS の商談では実質必須。未取得なら取得を優先。
4. 専業対比の機能深度（例：カオナビの配置シミュレーション、Carely の健診管理に相当する機能がない）。

## 3. 社会動向を踏まえて追加すべき機能（優先順）

1. **カスハラ/ハラスメント相談窓口パッケージ** 🔴 締切最短
   改正労働施策総合推進法により **2026年10月からカスハラ対策が全企業に義務化**（方針明確化・相談体制・事後対応・抑止措置の4本柱）。既存の consultation（相談窓口）基盤を拡張し、相談記録・対応フロー・方針掲示・AI 生成研修（eラーニング連動）をパッケージ化する。少ない開発量で「義務化対応」を訴求できる。
2. **50人未満事業場向けストレスチェック簡易モード** 🔴 中核強化
   労働安全衛生法改正により **2028年4月から50人未満事業場も義務化**（対象は数十万事業場）。2026年2月公表の「小規模事業場ストレスチェック制度実施マニュアル」（厚労省）に準拠した簡易実施モード・実施事務従事者支援・複数事業場/グループ横断管理を追加。最大の資産で最大の新市場を取る。
3. **外部連携＋公開 API**
   freee / マネーフォワード / SmartHR との従業員データ同期、Slack / LINE WORKS 通知。「労務は他社」戦略の生命線。
4. **勤怠コンプライアンス・アラート**
   労基法大改正（勤務間インターバル11時間・連続勤務13日上限等）は2026年国会提出見送りだが2027年以降に施行見込み。勤怠データは既にあるため、36協定分析にインターバル/連勤アラートを先行実装し「法改正に先回りしている製品」として差別化。
5. **AI エージェント化**
   2026年 HR テック最大のトレンド（採用リーダーの52%が採用 AI エージェント導入予定）。既存の hr-assistant / RAG を「相談に答える」から「申請処理・面接日程調整・スクリーニングを実行する」対話型エージェントへ発展させる。
6. **人的資本レポート自動生成**
   開示義務は大企業だが、取引先・金融機関からの開示要請が中小に波及中。hr-kpi / executive-summary の集計基盤を活かして経営者向けレポートを自動生成する。
7. **その他**
   育児介護休業法対応（子の看護等休暇・意向聴取記録）、休職・復職管理（Carely 対抗）、サーベイの多言語配信、PWA 化。

## 4. マーケティング戦略（一人会社前提）

開発・営業・サポートを一人で担う前提では、「人が動かないと売れない・回らない」仕組みは最初から捨てる。基本方針は **PLG（プロダクト主導の成長）× 法改正ドリブンのコンテンツ SEO × 社労士パートナー網** の3本柱。営業人員の代わりにプロダクトとコンテンツと他人の営業網に売らせる。

### 4-1. 戦略の前提（一人会社の制約から導かれる原則)

| 制約                                     | 導かれる原則                                                                                                               |
| ---------------------------------------- | -------------------------------------------------------------------------------------------------------------------------- |
| 商談・訪問営業に割ける時間がない         | セルフサーブ完結（申込→初期設定→決済まで無人）。Stripe＋フリーミアムは実装済みで、足りないのは「初期設定ウィザード」       |
| サポート人員がいない                     | ドキュメント＋RAG チャットボット（inquiry-chat）でサポートを自動化。自社サポートに自社 AI を使う事例自体が営業材料になる   |
| 広告予算が限られる                       | ビッグワード（「人事システム」等）で SmartHR と SEO 勝負はしない。法改正×義務化のロングテールキーワードに集中              |
| 信頼性資産（実績・認証・会社規模）がない | 「会社」ではなく「専門性」で信頼を作る。法改正解説コンテンツ・無料診断ツール・導入事例（初期は無償提供と引き換えでも取る） |
| 全方位の機能訴求は回らない               | 訴求は1つの楔（ウェッジ）に絞る。楔＝ストレスチェック（法定義務・締切あり・検索需要が明確）                                |

### 4-2. 柱①：法改正ドリブンのコンテンツ SEO（既存チャネルの集中運用）

既存チャネル（hr-dx.jp／blog.hr-dx.jp／Note／X）を「法改正の一次情報を中小企業向けに最速で噛み砕くメディア」に一本化する。

- **狙うキーワード群（締切が近い順）**
  - カスハラ対策 義務化 いつから／中小企業 対応（〜2026年10月）
  - ストレスチェック 50人未満 義務化／小規模事業場 マニュアル（〜2028年4月）
  - 高ストレス者 面談 流れ／労基署 報告書 書き方（通年の実務検索）
- **記事→リードの動線**：記事末に「無料でできる」導線を必ず置く。①ストレスチェック無料実施（フリーミアム）②カスハラ対応規程・体制チェックリスト等の**テンプレート配布**（メールアドレス獲得）③「義務化対応 診断」ミニツール
- **X／Note の役割分担**：X は法改正ニュースの速報＋開発の裏側（一人開発は透明性がむしろ差別化になる）、Note は導入ストーリー・思想（2大ゴール）を語る場、ブログは SEO 記事の本体
- コンテンツ制作自体を AI（本製品の Gemini 基盤）でパイプライン化し、週次の投稿を仕組みで回す

### 4-3. 柱②：社労士・産業医パートナー網（営業のアウトソース）

一人会社の営業力不足を埋める最重要チャネル。中小企業はツールを「検索して買う」より「顧問社労士に勧められて買う」ことが多い。

- **社労士向けパートナープラン**：顧問先を束ねて管理できるマルチテナント管理画面（実装済みの saas_adm 基盤を流用）＋紹介手数料 or 卸価格。社労士にとってはカスハラ・ストレスチェック義務化が顧問先への提案ネタになるため、利害が一致する
- **産業医紹介会社・EAP 事業者との連携**：company_doctor（産業医面談フォロー）機能は産業医側の業務ツールとしても訴求でき、彼らの顧客網に乗れる
- 商工会議所・自治体の中小企業支援施策（IT導入補助金の対象ツール登録を含む）も、一人でも申請ベースで乗れる販路

### 4-4. 柱③：PLG（プロダクトに売らせる）

- **フリーミアムの再設計**：現在の「AI 月10回」制限は楔として弱い。「従業員30名まで or 年1回のストレスチェックは無料」のように、**法定義務を無料で1回体験できる**設計にし、集団分析・産業医連携・通年のパルス／離職リスクを有料化する
- **セルフオンボーディング**：CSV 一括投入→組織図→初回ストレスチェック配信までを無人で完了できる初期設定ウィザード（onboarding/QuickStart の拡張）。「IT 部門がなくても 30 分で始められる」を売りにする
- **バイラルループ**：ストレスチェック・サーベイは全従業員がメールで受け取る製品特性上、受検画面のフッター「Powered by HR-DX」が最安の広告になる（無料プランのみ表示）
- **比較サイト掲載**：BOXIL・ITトレンド・アスピック等への掲載は一人でも申請可能で、検討土俵に乗るための最低条件

### 4-5. やらないこと（一人会社の防御）

- 展示会出展・テレアポ・訪問営業（時間単価が合わない）
- ビッグワード SEO・大手比較キーワードでの広告出稿（資金勝負になる）
- 楔（ストレスチェック／組織健康）以外の機能を主語にした発信（タレマネ・採用は「入った後に気づかせる」アップセル要素と位置づける）
- 電話サポート・個別カスタマイズ対応の約束（スケールしない負債になる）

### 4-6. 実行ロードマップと KPI（目安）

| 時期      | 施策                                                                                                 | KPI                          |
| --------- | ---------------------------------------------------------------------------------------------------- | ---------------------------- |
| 〜2026/09 | カスハラ義務化コンテンツ集中投下＋チェックリスト配布、比較サイト掲載、初期設定ウィザード             | 記事経由リード数、無料登録数 |
| 2026/10〜 | カスハラ施行に合わせた相談窓口パッケージ投入（機能優先度1と連動）、社労士パートナー 5〜10 事務所開拓 | パートナー経由テナント数     |
| 2027年    | 50人未満義務化の先回りコンテンツ＋「小規模事業場向け無料ストレスチェック」告知開始、導入事例3件公開  | 無料→有料転換率、解約率      |
| 2028/04〜 | 義務化施行の需要ピークを刈り取り                                                                     | 有料テナント数、MRR          |

**要点：一人会社の hr-dx にとってマーケティングとは「広告」ではなく、①締切のある法改正需要に最速でコンテンツを当てる、②社労士の営業網を借りる、③プロダクト自身に無人で売らせる——の3つを仕組みとして積み上げること。** これは機能追加の優先度（4-6 と第3章の1・2が連動）とも整合する。
